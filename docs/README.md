# Documentation

This directory contains development and historical documentation for the Zuo Budget Tracker.

## Contents

### Development History

- **FEATURES_IMPLEMENTED.md** - Comprehensive list of implemented features and their specifications
- **ISSUES_FIXED.md** - Record of bugs fixed and solutions implemented
- **REFACTORING_SUMMARY.md** - Summary of major code refactoring efforts

### Feature Documentation

- **ACTIONS_COLUMN_REMOVED.md** - Documentation of the actions column removal and its rationale
- **BANK_RECONCILIATION_UPDATES.md** - Bank reconciliation feature updates
- **TRENDS_IMPLEMENTATION.md** - Implementation details for the trends analysis feature

## For Users

For user-facing documentation and setup instructions, see the main [README.md](../README.md) in the project root.

## For Developers

These documents provide context about:
- Why certain design decisions were made
- How features evolved over time
- Solutions to complex problems encountered during development
- Lessons learned from refactoring efforts

When adding new features or making significant changes, consider documenting:
- The problem being solved
- Design decisions and trade-offs
- Implementation approach
- Testing strategy
- Known limitations or future improvements
